# SchoolVision

**SchoolVision** — это веб-приложение для школы, которое объединяет 3D-модель здания, систему подачи заявок и административный реестр для их обработки. Пользователь может открыть 3D-модель школы, выбрать точку прямо на модели и отправить заявку о проблеме. Администрация видит все заявки в одном реестре, меняет их статус и при необходимости открывает место проблемы в 3D.

## Что умеет проект

### Для пользователя
- выбрать этаж или общую модель школы;
- открыть 3D-модель в браузере;
- кликнуть по конкретной точке на модели;
- автоматически передать координаты в форму заявки;
- заполнить тему, описание и категорию;
- отправить заявку в базу данных.

### Для администрации
- войти в админ-панель;
- видеть полный реестр заявок;
- просматривать название, описание, автора, статус и координаты;
- менять статус заявки;
- удалять заявку;
- переходить к месту проблемы через 3D-просмотр.

## Логика работы системы

### Алгоритм работы пользователя
1. Пользователь открывает `map.php`.
2. Выбирает нужный уровень: вся школа, 1 этаж, 2 этаж, 3 этаж.
3. Переходит в `viewer.php`, где загружается выбранная 3D-модель.
4. Кликает по месту на модели.
5. Скрипт `viewer.js` с помощью `Raycaster` определяет координаты точки `(x, y, z)`.
6. Координаты сохраняются и пользователь попадает на страницу `infoproblem.php`.
7. На странице формы пользователь указывает тему, описание и категорию.
8. Файл `save_issue.php` записывает заявку в таблицу `issues`.

### Алгоритм работы администрации
1. Администратор входит в систему под аккаунтом с ролью `admin`.
2. Нажимает кнопку **«Реестр заявок»** и переходит в `adminpanel.php`.
3. В реестре отображаются все заявки из базы данных.
4. Для каждой заявки администратор может:
   - изменить статус;
   - удалить заявку;
   - открыть место проблемы в 3D.
5. После обработки проблема переводится в нужный статус: `pending`, `in_work`, `resolved` или `rejected`.

## Архитектура проекта

Проект построен по классической клиент-серверной схеме.

### Frontend
Frontend отвечает за интерфейс и работу 3D:
- `HTML` — структура страниц;
- `CSS` — оформление;
- `JavaScript` — логика интерфейса;
- `Three.js` — отображение и управление 3D-моделью.

### Backend
Backend отвечает за авторизацию, обработку форм и запись в БД:
- `PHP` — серверная логика;
- `MySQL` — хранение данных;
- `PDO` — безопасное подключение к базе данных.

## База данных

### Таблица `users`
Хранит данные пользователей системы.

| Поле | Назначение |
|---|---|
| `id` | уникальный идентификатор пользователя |
| `nickname` | имя или ник пользователя |
| `email` | электронная почта |
| `password` | пароль |
| `role_id` | роль пользователя |

### Таблица `roles`
Определяет роли в системе.

| id | Роль |
|---|---|
| `1` | `user` |
| `2` | `admin` |

### Таблица `categories`
Хранит категории заявок.

Примеры категорий:
- поломки в кабинетах;
- техника;
- освещение;
- санитарные проблемы;
- другое.

### Таблица `issues`
Главная таблица проекта, в которой хранятся заявки.

| Поле | Назначение |
|---|---|
| `id` | уникальный идентификатор заявки |
| `user_id` | автор заявки |
| `category_id` | категория |
| `title` | тема заявки |
| `description` | описание проблемы |
| `status` | текущий статус |
| `x` | координата X |
| `y` | координата Y |
| `z` | координата Z |
| `created_at` | дата создания |

## Как работает клик по 3D

В `viewer.js` используется `Raycaster`. Он позволяет выпустить луч из камеры в сторону курсора и определить, в какую точку 3D-модели кликнул пользователь.

После клика:
1. определяется точка пересечения;
2. создаётся визуальный маркер;
3. координаты сохраняются;
4. открывается форма подачи заявки.

Это позволяет отказаться от ручного выбора кабинетов и перейти к точной привязке к месту на модели.

## Как загрузить проект на GitHub

### Вариант 1. Быстро через сайт GitHub
Загрузить один `.zip` файл можно, но это плохой вариант для проекта.

Почему это неудобно:
- GitHub не покажет структуру проекта как код;
- нельзя нормально смотреть изменения по файлам;
- неудобно редактировать README;
- проект будет выглядеть как один архив, а не как репозиторий.

Правильно — загружать **сами файлы и папки**, а не архив.

### Вариант 2. Правильный способ через Git
1. Создай новый репозиторий на GitHub.
2. Открой папку проекта в терминале.
3. Выполни команды:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/USERNAME/REPOSITORY.git
git push -u origin main
```

Здесь:
- `USERNAME` — твой логин GitHub;
- `REPOSITORY` — имя репозитория.

### Вариант 3. Если не хочешь работать через консоль
Можно открыть репозиторий на GitHub и вручную перетащить в него:
- файлы `.php`;
- `viewer.js`;
- `README.md`;
- папку `Prefabs/` (если модели не слишком большие).

Главное — **не загружать только zip**, а именно распакованный проект.

## Как запустить проект локально

1. Помести проект на локальный сервер:
   - OpenServer;
   - XAMPP;
   - Beget;
   - другой PHP-хостинг.

2. Создай базу данных в phpMyAdmin.

3. Настрой файл `connect.php`.

Пример:
```php
<?php
$pdo = new PDO(
    "mysql:host=localhost;dbname=schoolvision;charset=utf8mb4",
    "root",
    "",
    [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC
    ]
);
```

4. Создай нужные таблицы:
- `users`
- `roles`
- `categories`
- `issues`

5. Загрузить `.glb` модели в `Prefabs/`.

6. Открыть в браузере:
- `map.php`
- `viewer.php`
- `adminpanel.php`
